# Донорский кошелёк

Экран Donor Wallet — это стартовая точка всего AntiDrain **Recovery flow**.

Если сделать этот шаг небрежно, все следующие шаги станут рискованнее.

## Для чего нужен donor wallet

Donor wallet — это отдельный кошелёк, который AntiDrain использует, чтобы:

- держать нативную монету для gas
- покрывать staged execution
- отправлять финальные подготовленные транзакции

Он специально отделён от compromised wallet.

## Два валидных способа начать

### Вариант 1: создать новый donor wallet локально

Подходит, если вы хотите получить чистый donor wallet прямо внутри текущей browser session.

### Вариант 2: импортировать свой donor wallet

Подходит, если у вас уже есть кошелёк, который вы осознанно хотите использовать как donor.

Импорт возможен через:

- donor private key
- donor mnemonic

## Что означает каждое поле на экране

### Wallet address

Это публичный адрес donor wallet.

Он нужен, чтобы:

- проверить, какой donor wallet сейчас активен
- при необходимости пополнить donor wallet снаружи
- сверить его с вашими записями

Кнопка `Copy` становится полезной только после того, как кошелёк реально существует.

### Private key

Это приватный ключ donor wallet.

Важное поведение:

- поле по умолчанию заблокировано
- для ручного изменения нужно осознанно включить режим редактирования
- если вы вводите туда текст, donor state считается меняющимся, пока ввод снова не станет валидным

### Mnemonic

Это recovery phrase donor wallet.

Важное поведение:

- поле тоже по умолчанию заблокировано
- для ручного изменения нужно нажать `Edit`
- редактирование мнемоники — это high-risk operation, потому что она может полностью сменить donor wallet context

### Generate Wallet

Создаёт новый donor wallet локально в текущей browser session.

### Download Wallet

Экспортирует **Backup JSON** donor wallet.

Это не декоративная кнопка. Именно этот шаг разблокирует следующий этап.

## Что лежит внутри Backup JSON

Экспортированный **Backup JSON** donor wallet содержит секреты donor wallet.

Практически это означает, что там лежат:

- donor address
- donor private key
- donor mnemonic
- security notice

Хранить этот файл нужно как полноценный секрет.

## Почему сайт просит подтвердить backup

После скачивания UI просит подтвердить, что вы действительно обработали **Backup JSON**.

Это нужно потому, что многие пользователи нажимают download и потом уже не знают, куда ушёл файл.

Приложение хочет явное подтверждение от оператора, прежде чем открыть следующие шаги.

## Что происходит при ручном вводе

Если вы вручную редактируете donor inputs:

- текущий donor wallet state может стать временно pending или invalid, пока ввод не станет корректным
- address display может очиститься, пока приложение ждёт валидный input
- copy и edit labels меняются в зависимости от того, пустое поле или в edit-mode

Это сделано специально. Приложение не хочет делать вид, что полу-введённый секрет уже является валидным кошельком.

## Почему lock на secret fields — это хорошо

Блокировка edit-mode существует, чтобы уменьшить риск случайных изменений.

Она помогает избежать ошибок вроде:

- вставки секрета не в то поле
- случайного редактирования уже валидного donor secret
- перезаписи корректного donor wallet частичным текстом

## Прогресс-карточка и readiness states

Блок справа отвечает на один вопрос:

**Можно ли уже безопасно идти на следующий этап?**

Он отслеживает:

- существует ли donor wallet
- обработан ли backup step
- можно ли разблокировать Actions

## Поведение после refresh

Сайт умеет восстановить donor wallet из session текущей вкладки после обычного refresh.

Это полезно, но не должно быть вашим главным backup-механизмом.

Экспортированный **Backup JSON** по-прежнему остаётся основным надёжным fallback.

## Хороший способ пройти этот экран

1. создать или импортировать donor wallet
2. проверить donor address
3. нажать `Download Wallet`
4. сохранить **Backup JSON** в безопасное место
5. убедиться, что файл реально читается
6. только после этого переходить в Actions

## Частые ошибки новичков

### Ошибка 1: считать donor wallet временным и поэтому неважным

Donor wallet всё равно остаётся реальным кошельком, на котором могут лежать реальные средства.

### Ошибка 2: пропустить или "наполовину" сделать шаг с **Backup JSON**

Именно поэтому следующий этап не открывается, пока backup не обработан.

### Ошибка 3: путать donor и compromised secrets

Никогда не вставляйте donor secrets в поля, предназначенные для compromised wallet.

### Ошибка 4: изменить поле и забыть, что этим вы поменяли wallet context

Если вы вручную правили secret, обязательно проверьте, что итоговый donor address всё ещё тот, который вы хотели видеть.

## Checklist перед выходом с этого экрана

Перед тем как уйти с Donor Wallet, убедитесь, что:

- donor address именно тот, который вы хотели использовать
- donor private key или mnemonic валидны
- **Backup JSON** сохранён
- вы понимаете, где лежит backup-файл
