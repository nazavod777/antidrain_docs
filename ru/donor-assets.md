# Активы донора

`Donor Assets` — это экран cleanup для donor wallet после основного **Recovery flow**.

Очень многие пользователи настолько сосредотачиваются на compromised wallet, что потом забывают проверить сам donor wallet. Этот экран нужен именно затем, чтобы не допустить такую ошибку.

## Что можно сделать на этом экране

Здесь можно:

- выбрать сеть для donor cleanup
- обновить donor native balance
- обновить donor ERC-20 balance
- вывести остатки native funds
- вывести остатки ERC-20

## Почему на donor wallet вообще могут остаться средства

После основного recovery flow на donor wallet могут лежать:

- остатки нативной монеты
- неиспользованный gas buffer
- ERC-20 токены
- остатки после staged execution

Проверка и cleanup donor wallet — это нормальная часть завершения **Recovery flow**.

## Верхние controls: network и custom RPC

Даже donor cleanup требует правильного network context.

Почему:

- native balances зависят от сети
- ERC-20 balances зависят от сети
- для token withdrawal всё равно нужен правильный chain и gas context

При необходимости можно использовать и custom RPC URL.

## Вывод нативной монеты

Обычный порядок такой:

1. выбрать сеть
2. обновить native balance
3. ввести `Recipient Address`
4. выполнить native withdrawal

### Важная деталь

Приложение не пытается blindly отправить весь нативный баланс целиком.

Сначала оно оценивает gas, резервирует стоимость газа и только затем отправляет остальной transferable remainder.

Если остатка слишком мало даже для оплаты своего же газа, приложение прямо об этом скажет.

## Вывод ERC-20

Обычный порядок такой:

1. выбрать сеть
2. ввести token contract address
3. обновить token balance
4. ввести `Recipient Address`
5. вывести токены

### Важное напоминание для новичка

Даже если вы выводите только ERC-20, donor wallet всё равно нужен нативный баланс для газа.

Нет нативной монеты — нет token withdrawal transaction.

## Что означает каждый input

### Token Contract Address

Это адрес ERC-20 контракта того токена, который вы хотите вывести с donor wallet.

Это не адрес получателя.

### Recipient Address

Это адрес, который должен получить donor cleanup withdrawal.

Перед отправкой его обязательно нужно перепроверять.

## Почему кнопки могут оставаться disabled

Native или token withdrawal могут быть заблокированы по нормальным причинам.

Типичные примеры:

- donor wallet ещё недоступен
- сеть не выбрана
- custom RPC отсутствует или невалиден в custom-mode
- token contract не указан или невалиден
- баланс ещё не обновлён
- `Recipient Address` невалиден
- другой withdrawal уже выполняется

## Почему refresh действительно важен

Экран хочет видеть свежий balance snapshot перед включением withdrawal.

Это полезно, потому что:

- баланс мог измениться после предыдущих шагов
- gas reserve может оказаться не таким, как вы ожидали
- token balance может уже быть нулевым, хотя вы думали, что там что-то осталось

## Пограничный случай для native: слишком мало на gas

Такое часто бывает с совсем маленькими остатками.

Если native balance слишком мал даже для оплаты собственного withdrawal gas, страница заблокирует операцию.

Это не баг. Это значит, что amount ниже практического порога отправки.

## Пограничный случай для токена: нулевой balance

Если обновлённый token balance равен нулю, приложение честно остановится на этом месте и не будет делать вид, что отправка возможна.

## Хорошая cleanup-привычка

После того как основной **Recovery flow** завершился:

1. откройте Donor Assets
2. проверьте native balance
3. проверьте нужные ERC-20 leftovers
4. при необходимости выведите их
5. только после этого считайте donor cleanup завершённым

## Частые ошибки

### Ошибка 1: забыть, что donor wallet тоже нужен gas

Для token withdrawal всё равно нужен нативный gas.

### Ошибка 2: использовать не тот token contract

Если адрес token contract неверный, и balance snapshot тоже будет неверным.

### Ошибка 3: пропустить refresh

Всегда обновляйте баланс перед тем, как считать displayed value текущим.

## Финальный checklist

Перед donor cleanup withdrawal убедитесь, что:

- сеть выбрана правильно
- token contract верный, если он нужен
- recipient правильный
- balance snapshot свежий
- вы понимаете, стоит ли этот amount вообще двигать
