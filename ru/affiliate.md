# Партнёрская ссылка

Страница `/affiliate` помогает создать ссылку с вашим EVM-адресом для выплат.

Если человек открыл AntiDrain по вашей ссылке и потом использовал поддерживаемый платный rescue-flow, часть комиссии может уйти на ваш адрес.

## Как создать ссылку

1. Откройте `/affiliate`.
2. Введите EVM-адрес для выплат.
3. Скопируйте ссылку.
4. Передайте её пользователю.

## Требования к affiliate wallet

Адрес для выплат должен быть обычным EVM-кошельком, которым вы контролируете.

Не используйте адрес, если:

- это smart contract;
- на адресе есть активная EIP-7702 delegation;
- адрес сейчас показывает delegated code.

Если вы не уверены, проверьте адрес в explorer или через RPC перед тем, как создавать ссылку. Если на адресе есть EIP-7702 delegation, сначала снимите её через Remove Delegation, а уже потом используйте адрес как affiliate wallet.

## Как это работает

Ссылка содержит ваш адрес в параметре `?affiliate=`.

Когда пользователь открывает сайт по такой ссылке, AntiDrain сохраняет ваш адрес локально в браузере пользователя как affiliate wallet.

Если в браузере пользователя уже был сохранён другой affiliate wallet, новый адрес обычно не заменяет старый. Пользователь может очистить сохранённый адрес на странице `/affiliate`.

Партнёрская ссылка не увеличивает общую комиссию пользователя. Она только меняет то, как уже существующая комиссия делится между protocol и affiliate.

## Проценты и реварды

Есть два типа выплат.

### 1. Token rescue fee

Это комиссия в токене, который спасается.

Когда поддерживаемый token rescue-flow применяет стандартную token fee:

- всего берётся `20%` от спасаемой суммы;
- пользователь/получатель получает `80%`;
- protocol получает `15%`;
- affiliate получает `5%`.

Если активного affiliate wallet нет:

- пользователь/получатель получает `80%`;
- protocol получает `20%`;
- affiliate получает `0%`.

Примеры:

- если спасается `100 USDT`, affiliate получает `5 USDT`;
- если спасается `1,000 TOKEN`, affiliate получает `50 TOKEN`;
- если спасается `0.5 ETH` через поддерживаемый native transfer-all flow с таким же split, affiliate получает `0.025 ETH`.

### 2. Native service fee

Это отдельная комиссия в нативной монете сети: ETH, BNB, POL, AVAX и так далее.

Сайт считает её в fee units. Один fee unit равен `$5`, а затем переводится в нативную монету по текущей цене.

Если активен affiliate wallet:

- protocol получает `75%` native service fee;
- affiliate получает `25%` native service fee.

То есть affiliate получает `$1.25` за каждый fee unit.

Примеры:

- если service fee равен `$5`, affiliate reward равен `$1.25` в нативной монете;
- если service fee равен `$10`, affiliate reward равен `$2.50` в нативной монете;
- если service fee равен `$15`, affiliate reward равен `$3.75` в нативной монете.

Точная сумма в ETH/BNB/POL зависит от цены нативной монеты в момент расчёта.

## Когда affiliate получает выплату

Affiliate получает выплату только если одновременно выполняются все условия:

- пользователь открыл сайт по вашей ссылке;
- ваш EVM-адрес сохранился как affiliate wallet;
- пользователь дошёл до платного rescue-flow;
- выбранный flow поддерживает affiliate fee sharing;
- транзакция успешно отправилась и выполнилась;
- в этом flow реально была комиссия, которую можно разделить.

Affiliate может получить выплату в таких случаях:

- при поддерживаемом ERC-20 token rescue, где применяется token fee;
- при Permit Rescue, когда выбран режим с token fee;
- при DeBank/Bundler token transfer или native transfer-all, если конкретный route применяет fee split;
- при Custom Batch / DeBank действиях с native service fee, например raw call, NFT/pool действия или другие fee-bearing операции.

Affiliate не получает выплату, если:

- пользователь не открыл вашу ссылку;
- адрес не сохранился или был очищен;
- flow бесплатный;
- flow не поддерживает affiliate fee sharing;
- транзакция не была отправлена или упала до выплаты;
- affiliate wallet не может принять выплату.

## Важно

Партнёрская ссылка не меняет безопасность rescue-сценария.

Если affiliate wallet не может принять native payout или token payout, rescue execution не должен ломаться. В этом случае affiliate reward может уйти по fallback-логике, обычно protocol recipient получает эту часть.

## Что проверять

Проверяйте, что:

- адрес введён правильно;
- адрес не является smart contract;
- адрес не имеет активной EIP-7702 delegation;
- ссылка скопирована полностью;
- пользователь открыл именно вашу ссылку;
- на странице `/affiliate` у пользователя отображается нужный сохранённый affiliate wallet;
- flow действительно поддерживает fee sharing;
- в выбранном flow есть комиссия, из которой может быть выплата.
